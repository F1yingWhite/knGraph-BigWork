这是图片中关于第二个作业（即“课程设计”）的文字内容：

**课程设计：贡献一个开放知识图谱到 [OpenKG.cn](http://OpenKG.cn) （鼓励）**

- 三人或四人一组，主要面向开放域数据，要求深度利用抽取技术；
- 基于 cnSchema 进行扩展；
- 推荐进一步实现完整的应用形式，如搜索、QA 或挖掘分析等；

## 什么是 cnSchema？

**cnSchema**（全称：Chinese Open Knowledge Graph Schema）是一个**面向中文开放知识图谱的 Schema（模式/本体）标准**。

简单来说，如果把知识图谱比作一个巨大的数据库，cnSchema 就是这个数据库的“蓝图”或“规范手册”，规定了数据应该如何分类、如何描述属性以及如何建立关联。

以下是关于 cnSchema 的详细解释，特别是针对你作业背景（OpenKG）的解读：

### 1\. 它的核心定义

cnSchema 是由 **OpenKG 社区**发起并维护的。它的目的是解决中文领域知识图谱**数据标准不统一**的问题。

- **它是做什么的？** 它定义了常见的**类（Class）**（例如：人物、地点、组织、电影）和**属性（Property）**（例如：出生日期、位于、导演是）。
- **它的基础：** 它并没有凭空创造一套新标准，而是主要复用了国际通用的 **Schema.org** 标准，并结合中文语境进行了扩展。

### 2\. 为什么会有 cnSchema？

虽然国际上已经有 Schema.org，但在处理中文特定数据时往往不够用。cnSchema 做了两件事：

1.  **本地化（本土化）：** 增加了中国特有的概念。例如：中国古代的朝代、中医概念、中国菜系、特定的行政区划等，这些在西方标准中往往是缺失的。
2.  **上层映射：** 它确保你构建的图谱能和 Wikidata 或 Schema.org 对应上，方便以后数据的互联互通。

### 3\. 结合你的作业：“基于 cnSchema 进行扩展”

你图片中的课程设计要求是：“贡献一个开放知识图谱……基于 cnSchema 进行扩展”。这句话的意思是，老师不希望你们从零开始瞎编一套分类体系。

**具体操作建议：**

1.  **不要重复造轮子：** 如果你要构建一个关于“三国演义”的图谱，首先去 cnSchema 查一下，有没有“Person（人）”这个类？有。那就直接用，不要自己创建一个叫 `RenWu` 的类。
2.  **扩展特有内容：** 如果 cnSchema 里没有你需要的特定概念（比如“三国武将”这个细分职业），你可以在 cnSchema 的“Person”类下面，**继承并扩展**出一个子类叫“General（武将）”。

### 4\. 总结：它长什么样？

cnSchema 通常支持 JSON-LD 或 RDF 格式。一个简单的例子看起来像这样：

```json
{
  "@context": "http://cnschema.org",
  "@type": "Person", // 使用了 cnSchema 定义的 Person 类
  "name": "鲁迅",
  "jobTitle": "作家",
  "alumniOf": "仙台医科专门学校" // 使用了 cnSchema 定义的属性
}
```

**一句话总结：**
cnSchema 就是**中文知识图谱界的“普通话”**。你的作业要求使用它，是为了保证你做出来的图谱别人也能看懂，并且能融入 OpenKG 的大家庭。
